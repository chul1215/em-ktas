# PRD: 응급실 KTAS 검사처방 분석 대시보드

> **문서 버전:** v2.0
> **작성일:** 2026-03-07
> **프로젝트명:** EM-KTAS Dashboard
> **목적:** 대전선병원·유성선병원 응급실 KTAS 등급별 검사처방 현황, KPI 달성도, 수익 분석을 한눈에 파악할 수 있는 대시보드

---

## 1. 프로젝트 개요

### 1.1 배경
- 양 병원 응급실의 KTAS 등급별 기본lab 처방율, 의사별 처방 편차, 수익 구조를 통합 모니터링할 필요성
- 신규 KPI 체계(병원별 차등: 대전 100/95/80/50/15이하, 유성 95/95/80/50/15이하) 적용에 따른 목표 대비 현황 실시간 추적
- ROOMi(적정진료관리팀 AI)의 분석 견해를 현장에 전달하기 위한 인터페이스

### 1.2 사용자
- 응급실 운영 관리자
- 적정진료관리팀
- 병원 경영진

### 1.3 기술 스택
- **프론트엔드:** HTML + CSS + JavaScript (단일 파일 SPA)
- **차트 라이브러리:** Chart.js 4.4.4 (CDN)
- **데이터:** `D` 객체 내 하드코딩 (정적 데이터 기반, 별도 백엔드 없음)
- **배포:** GitHub Pages (root `index.html` → `dashboard/index.html` 리다이렉트)

## 5. 인터랙션 명세

### 5.1 병원 전환 스위치
- 토글 클릭 시 전체 대시보드 데이터가 해당 병원으로 전환
- `renderAll()` 호출로 모든 섹션 재렌더링
- 전환 시 fadeIn 애니메이션 (300ms)
- 활성 버튼: blue gradient 배경 + 흰색 텍스트

### 5.2 차트 인터랙션
- 호버: 다크 테마 툴팁 (`rgba(10,15,35,0.9)` 배경, 흰색 텍스트, `rgba(255,255,255,0.15)` 테두리)
- Chart.js 인스턴스는 `charts` 객체로 관리, 전환 시 `.destroy()` 후 재생성

### 5.3 테이블 인터랙션
- 행 호버: `rgba(255,255,255,0.05)` 배경 하이라이트

### 5.4 탭 전환
- Pill 스타일: 활성 탭은 blue gradient 배경 + inset 하이라이트
- 탭 전환 시 콘텐츠 영역 display 토글 (`classList.add/remove('active')`)

### 5.5 카드 인터랙션
- 호버: `translateY(-3px)` + `var(--glass-shadow-hover)` + `var(--glass-bg-hover)`

## 11. KPI 재설계 — 3차 변경 적용 완료

KPI 3차 변경이 반영되었다. 주요 변경 사항:

**2차 변경 (이전):**
- KPI 목표 변경: KTAS 3 (93%→96%), KTAS 4 (75%→80%), KTAS 5 (20%→30%)
- 수익 시나리오: 3종 → 2종(외래/입원연계)으로 단순화
- 수술시(347,550원) 시나리오 폐기 → 병원별 입원연계 단일 기준으로 통합

**3차 변경 (현재):**
- KPI 목표를 병원별로 분리 — 기존 양 병원 공통 → 병원별 차등
- 대전: {1:100%, 2:95%이상, 3:80%이상, 4:50%, 5:15%이하}
- 유성: {1:95%이상, 2:95%이상, 3:80%이상, 4:50%, 5:15%이하}
- KTAS 5 기준 방향 전환: "이상(floor)" → "이하(cap)" — 낮을수록 적정
- 외래 KPI 수익: (v - ac) × 병원별 KPI% × 기본lab단가
- 입원 KPI 수익: ac × 100% × 입원추가단가
- 병원별 단가: 유성(158,570 / 514,830원), 대전(144,040 / 333,020원)
